How to Use This Pack
Before You Start
Read this page first. These 7 workflows are not prompts to run once. They are operating logic you install into your weekly routine.
Core principle
A workflow is worth keeping when it is: repeated (happens every week), structured (recognizable sequence), context-heavy (same background reused), and founder-touched (still depends on you too often). These 7 cover all four criteria.
1
Choose your Claude entry point
Each workflow works in Claude.ai (browser or app), Claude Code (terminal), or via API through n8n / Make / Zapier. The prompt is the same. The execution context changes. Start with claude.ai if you are new. Graduate to Claude Code once you want automation.
claude.ai Claude Code n8n Make Zapier
2
Set up your context file (CLAUDE.md)
For Claude Code users: create a CLAUDE.md file in your project root. Paste your company context, ICP, offer, and tone rules. Claude reads this automatically at the start of every session. This removes the re-explaining cost permanently.
CLAUDE.md
3
Fill in the placeholder values
Every prompt contains placeholders in [BRACKETS]. Replace these with your actual business context before running. The more specific your input, the more usable your output. Generic input = generic output.
4
Run weekly, not once
The ROI compounds with repetition. First run will save you 30 minutes. After 4 weeks of consistent use per workflow, you stop thinking about that work entirely. That is when leverage kicks in.
5
Connect to your tools only where repetition is real
Automation is optional. You do not need to connect all 7 workflows or build a master orchestration layer first. Start manual inside Claude. Only add a trigger for workflows you already run 3x per week or more. For most founders, that means inbox triage and meeting notes first. Proposal drafting, research briefs, content extraction, and weekly CEO review can stay manual for a long time without losing value.
Notion Gmail HubSpot Slack Linear
Model recommendation
Use Claude Sonnet for all 7 workflows as a starting point. It handles long-context inputs well and produces structured output reliably. For inbox triage at volume, Claude Haiku cuts token cost by 5-8x with acceptable quality. For anything requiring deep reasoning (decision briefs, CEO review), stay on Sonnet.
Automation note — keep it simple
You do not need a separate automation workflow for every workflow in this pack. Use the prompt manually first. Once one workflow proves useful, attach a single trigger to it. Example: new email -> Claude triage -> Slack or Notion, or transcript ready -> Claude summary -> CRM note. That is enough. The goal is fewer founder touches, not a more complex stack.
Workflow 01
Saves 2-3 hrs/week
Inbox Triage + Priority Routing
Stop scanning. Stop switching. Let Claude classify, prioritize, draft, and route every message before you touch it.
Inputs you provide
->Raw email dump or forwarded thread
->Your team member names + roles
->Your priority rules (optional)
->Open deals / active clients list
Claude outputs
->Priority bucket (P1 / P2 / P3 / FYI)
->2-line summary per message
->Recommended action + owner
->Reply draft (P1 only)
->Follow-up date flag
Use when
Monday morning inbox overload
Post-travel or post-weekend backlog
You have 3+ people sending you updates
You manage an active client pipeline
Skip when
Inbox has fewer than 10 messages
All messages are from 1 person
You already use a triage VA
Ready-to-use prompt - paste into Claude.ai or Claude Code
WORKFLOW_01 · inbox-triage.md
ROLE: You are my inbox triage assistant. You process incoming messages and produce a structured priority report. CONTEXT: - My name: [YOUR NAME] - My business: [1-2 sentence description] - Team members: [Name - Role, Name - Role] - Active clients: [Client A, Client B, Client C] - Active deals: [Deal name, stage] PRIORITY RULES: - P1 = Requires my response within 4 hours (client issue, revenue at risk, legal) - P2 = Requires response today (ongoing projects, warm leads, team blockers) - P3 = Requires response this week (general updates, cold outreach, FYI threads) - FYI = No action needed (newsletters, receipts, automated notifications) MESSAGES TO PROCESS: [PASTE ALL EMAILS HERE - include sender, subject, body] OUTPUT FORMAT: For each message: ## [Priority: P1/P2/P3/FYI] | [Sender] | [Subject] - Summary: [2 sentences max] - Action: [specific next step] - Owner: [me / team member name] - Due: [today / this week / no action] - Reply draft: [only for P1 messages] End with a SUMMARY TABLE showing total counts per priority bucket.

n8n / Make automation · optional
Gmail trigger: new email received
Claude node: run triage prompt
Notion output: priority inbox board
Slack alert: P1 messages only
No extra workflow needed: one Gmail trigger + one Claude step is enough

Claude Code setup
Create inbox.md with pasted emails
Run: claude "process inbox.md using triage rules"
Output saved to priority-report.md
Review takes under 5 minutes
Common mistakes
Pasting emails without context - Claude cannot prioritize without knowing your business and active deals
Asking Claude to reply to everything - only P1 replies save time; P2/P3 drafts create more editing work than writing them yourself
Running this once and stopping - value compounds when you do this every Monday and Thursday
Workflow 02
Saves 3-4 hrs/week
Meeting Notes into Action Systems
Most meetings fail in the handoff, not the room. This workflow turns raw notes or transcripts into a complete decision package in under 2 minutes.
Inputs you provide
->Raw meeting notes (messy is fine)
->Call transcript (Otter, Fathom, Fireflies)
->Attendee list with roles
->Meeting type (sales / ops / team / client)
Claude outputs
->What matters (3 bullets max)
->Decisions made with owners
->Blockers flagged with resolution path
->Next actions + due dates
->Follow-up email draft
->CRM / Notion update block
Ready-to-use prompt
WORKFLOW_02 · meeting-to-action.md
ROLE: You are my meeting processor. Turn raw notes into a structured action package. MEETING CONTEXT: - Meeting type: [sales call / team sync / client review / discovery] - Date: [DATE] - Attendees: [Name - Role, Name - Role] - Goal of meeting: [what we were trying to decide or accomplish] RAW NOTES / TRANSCRIPT: [PASTE NOTES OR TRANSCRIPT HERE] OUTPUT - produce all sections below: ## WHAT MATTERS [3 bullet points: most important outcomes or information from this call] ## DECISIONS MADE [Table: Decision | Owner | Deadline] ## BLOCKERS [Table: Blocker | Who needs to resolve | By when] ## NEXT ACTIONS [Table: Action | Owner | Due date | Priority] ## FOLLOW-UP EMAIL DRAFT [Subject: Re: [meeting name]] [Professional but direct email to all attendees summarizing decisions and actions] ## CRM / NOTION UPDATE [Single paragraph, past tense, ready to paste into deal notes or project log]

Common mistakes
Not specifying meeting type - a sales call and a team sync need completely different output structure
Sending the AI follow-up without review - always read the email draft before sending; Claude does not know your relationship dynamics
Skipping the blocker section - blockers that are not named in writing stay invisible and kill projects

Ready-to-use prompt
WORKFLOW_03 · proposal-draft.md
ROLE: You are my proposal writer. Turn discovery notes into a structured commercial proposal draft. MY BUSINESS: - Company: [YOUR COMPANY NAME] - Core offer: [what you sell in 1 sentence] - Pricing tiers: [Tier 1: $X - what included, Tier 2: $Y - what included] - Typical timeline: [e.g. 6-week engagement] CLIENT CONTEXT: - Client: [Company name, industry] - Team size: [N people] - Budget signal: [mentioned / not mentioned / range they gave] - Decision maker: [name and title] - Key pain they described: [paste their exact words if possible] DISCOVERY NOTES: [PASTE YOUR DISCOVERY NOTES HERE] OUTPUT: ## ENGAGEMENT SUMMARY [3-4 sentences restating their problem in their language. This is what they should feel is "exactly right".] ## PROPOSED SCOPE [Numbered list of deliverables with brief description each] ## WHAT IS NOT INCLUDED [Bulleted exclusions - prevents scope creep] ## TIMELINE [Week-by-week breakdown] ## INVESTMENT [Recommended tier with pricing rationale - 2 paragraphs] ## ASSUMPTIONS [What we are assuming to be true for this scope to hold] ## NEXT STEP EMAIL [Subject: Proposal - [Client Name] x [Your Company]] [Short email: context, link to proposal, one clear CTA]
Common mistakes
Using your language instead of the client's - paste their exact words from the call; Claude will mirror it and proposals convert better
Skipping the exclusions section - what is out of scope is as important as what is in scope
Sending without adding your tone - review the engagement summary and make it sound like you, not a template
Workflow 04
Saves 3-4 hrs/week
Outbound Personalization at Scale
You should not be writing each outbound message from zero. Give Claude the lead context and let it generate relevant, non-generic first messages.
Inputs you provide
->Lead name, company, role, LinkedIn URL
->Recent activity or trigger (post, funding, hire)
->Your ICP pain map
->Your offer and CTA
Claude outputs
->Lead pain hypothesis (1 paragraph)
->3 outbound message variants (different angle each)
->Objection-aware follow-up (day 3)
->Final bump (day 7)
Ready-to-use prompt
WORKFLOW_04 · outbound-personalization.md
ROLE: You are my outbound specialist. Write personalized first-touch messages and follow-up sequences. MY OFFER: - Product: [what you sell] - ICP: [who it is for, e.g. founders of 1-10 person B2B service companies] - Core pain solved: [specific pain, not generic] - Proof point: [specific result, e.g. saves 20h/week, closes 2x faster] - CTA: [what you want them to do, e.g. 15-min call / reply / sign up] LEAD CONTEXT: - Name: [FIRST NAME] - Company: [COMPANY] - Role: [TITLE] - Trigger / recent activity: [e.g. posted about hiring problems, raised seed round, recently hired] - LinkedIn URL: [URL] TONE RULES: - No em dashes. No curly quotes. No "I hope this finds you well." - Short paragraphs. Direct. First person. - Never mention AI, automation, or Claude in the message. - Sound like a founder, not a sales tool. OUTPUT: ## PAIN HYPOTHESIS [1 paragraph: what problem this lead is likely experiencing based on their role and trigger] ## MESSAGE VARIANT A (direct angle) [Under 80 words. Reference trigger. Lead with their pain. CTA at the end.] ## MESSAGE VARIANT B (curiosity angle) [Under 80 words. Open with a question. No pitch until line 3.] ## MESSAGE VARIANT C (social proof angle) [Under 80 words. Lead with a result you got for someone similar. Connect to them.] ## FOLLOW-UP DAY 3 [Under 50 words. Acknowledge no reply. Add one new piece of value or insight.] ## FOLLOW-UP DAY 7 (final bump) [Under 40 words. Low pressure close. Easy yes or no answer.]
Common mistakes
No trigger provided - generic outbound without a trigger reason gets ignored; always include what made you reach out now
Sending all 3 variants - pick one per lead; A/B test variants across batches of 20+ leads
Making Claude mention AI in the message - it kills trust instantly with founder audiences
Workflow 05
Saves 2-3 hrs/week
Research into Decision Briefs
Research only has value when it compresses decision time. Stop reading long documents and re-explaining findings. Let Claude produce the brief.
Inputs you provide
->Raw research, articles, or reports
->The decision you need to make
->Your constraints (budget, timeline, team)
->Options you are considering
Claude outputs
->Key findings (only what is decision-relevant)
->Trade-offs per option
->Risks with probability signals
->Clear recommendation with reasoning
->What to do now + what to ignore
Ready-to-use prompt
WORKFLOW_05 · research-to-brief.md
ROLE: You are my research analyst. Turn raw information into a decision-ready brief. No fluff. Output only what I need to decide. THE DECISION: [State the specific decision you need to make in one sentence. Example: Should we switch from HubSpot to Attio as our CRM?] MY CONSTRAINTS: - Budget: [max budget for this decision] - Timeline: [when this needs to be decided] - Team: [who is affected by this decision] - Non-negotiables: [things that cannot change regardless of what we decide] OPTIONS I AM CONSIDERING: [Option A: description] [Option B: description] [Option C: description (if any)] RESEARCH INPUT: [PASTE ALL RESEARCH, ARTICLES, NOTES, REVIEWS, DATA HERE] OUTPUT: ## KEY FINDINGS [Bullet list: only findings that change the decision. Ignore everything else.] ## OPTION COMPARISON | Criteria | Option A | Option B | Option C | [Fill in with relevant criteria based on the decision] ## RISKS [For each option: main risk + estimated probability (Low/Med/High) + mitigation] ## RECOMMENDATION [Single clear recommendation with 3-sentence rationale. Do not hedge.] ## WHAT TO DO NOW [The next 3 concrete actions to implement the recommendation] ## WHAT TO IGNORE [What looked important in the research but does not actually affect this decision]
Common mistakes
Framing the decision vaguely - "should we improve our sales process" is not a decision; "should we hire a full-time SDR or use Claude for outbound for the next 90 days" is
Not listing constraints - without constraints, Claude will give you a theoretically perfect recommendation you cannot implement
Treating the recommendation as final - use this brief as your starting point, not your ending point; validate the key assumption before committing

Ready-to-use prompt
WORKFLOW_06 · content-extraction.md
ROLE: You are my content strategist. Extract LinkedIn-ready content from my real operating material. No made-up frameworks. No generic advice. Only things that actually happened this week. MY AUDIENCE: - Platform: LinkedIn - Target reader: [e.g. founders of 1-20 person B2B service businesses] - My authority topic: [e.g. replacing manual founder work with AI workflows] - Tone: direct, operator-first, no generic AI educator language CONTENT FORMAT RULES (apply to all outputs): - No hashtags. No em dashes. No curly quotes. - Links only in first comment, never in post body. - Short paragraphs. 10-19 words each. 14+ paragraphs per post. - End with: "Repost if [specific founder situation]" or "Save this." - No banned words: leverage, optimize, seamless, delve, robust, unlock, journey RAW MATERIAL THIS WEEK: [PASTE: call transcripts, notes, decisions made, client questions, frameworks used, problems solved, things that surprised you, things that failed] OUTPUT: ## 5 POST HOOKS [Each hook: 2 lines only. Stop-the-scroll quality. Number them 1-5.] ## 2 CONTRARIAN ANGLES [What you believe that most people in your space get wrong. Based only on the material provided.] ## 1 FRAMEWORK BREAKDOWN [Name the framework you used this week. List the steps. This becomes a carousel or infographic.] ## 3 POST OUTLINES [For each: Hook + 6 paragraph topics + CTA. Do not write the full post. Outlines only.]
Common mistakes
Pasting nothing and asking Claude to "come up with content ideas" - Claude will generate generic output; your real operating material is the only source of genuine authority content
Using all 5 hooks in one week - pick the strongest 2; post quality beats post frequency on LinkedIn in 2026
Skipping the format rules in the prompt - without them, Claude defaults to hashtags and em dashes which the 360Brew algorithm penalizes
Workflow 07
Saves 2-3 hrs/week
Weekly CEO Review + Priority Reset
The most important 30 minutes of your week. This workflow runs a structured review against your goals, pipeline, and open loops so Monday has clarity instead of fog.
Inputs you provide
->This week's goal set (from last review)
->Pipeline status (deals, clients, MRR)
->Brain dump of what happened this week
->Biggest bottleneck you are aware of
Claude outputs
->What moved vs. what stalled (scored)
->Biggest risks with urgency flag
->Open loops with resolution path
->3 priorities for next week (ranked)
->3 decision prompts requiring your attention
Ready-to-use prompt - run every Friday or Sunday
WORKFLOW_07 · weekly-ceo-review.md
ROLE: You are my operating partner. Run my weekly CEO review. Be direct. Flag what is not working. Do not soften bad signals. BUSINESS CONTEXT (update weekly): - Current MRR: [$X] - MRR goal for this month: [$X] - Active clients: [N] - Open deals in pipeline: [N - stages and sizes] - Team size: [N] - This quarter's 3 goals: [Goal 1, Goal 2, Goal 3] THIS WEEK'S GOALS (set last Sunday): [paste last week's 3 priorities here] BRAIN DUMP - what happened this week: [write anything: wins, losses, conversations, decisions, surprises, things that stressed you, things that felt slow. Messy is fine.] BIGGEST BOTTLENECK RIGHT NOW: [the one thing slowing everything else down] OUTPUT: ## WEEK SCORECARD [Table: Goal | Status (Done/Partial/Missed) | Key reason] [Overall score: X/3 goals completed] ## WHAT MOVED [Bullet list: real progress made this week - be specific] ## WHAT STALLED [Bullet list: what did not move and why - be honest] ## TOP 3 RISKS [Risk | Urgency (High/Med/Low) | What happens if ignored | Suggested response] ## OPEN LOOPS [Anything unresolved that is taking mental space | Who needs to close it | By when] ## NEXT WEEK PRIORITIES 1. [Priority 1 - most important, tied to revenue or critical path] 2. [Priority 2] 3. [Priority 3] ## DECISION PROMPTS [3 questions you need to answer before end of next week to unblock progress]
Common mistakes
Skipping the brain dump - the value is not in the template, it is in the act of externalizing everything in your head so Claude can find patterns you cannot see while you are in it
Setting more than 3 priorities - 3 priorities is a forcing function; more than 3 means you have not decided what matters, which is the job Claude is helping you avoid
Running this on Monday instead of Friday - by Monday the fog is already present; run it Friday afternoon and start Monday with clarity already built
What to Do Next
These 7 workflows are enough
to create real leverage.
Not because Claude becomes your company. Because it stops you from acting like the manual router for every repeated decision.
The 3 costs these workflows remove
Re-explaining
Same context rebuilt from zero every time a task restarts or a new tool is involved.
Re-deciding
Same type of decision made by the founder again every week instead of running on rules.
Re-starting
Work breaking between steps, people, and tools with no continuity and no handoff logic.
1
Start with Workflow 07 (Weekly CEO Review)
Run it this Friday. It costs 30 minutes. The clarity it produces on Monday is worth more than any other workflow in this pack. It is also the one that reveals which of the other 6 you need most urgently.
2
Add one workflow per week for 6 weeks
Do not install all 7 at once. Pick the one that costs you the most time right now. Run it for a week. Then add the next. After 6 weeks you have a complete operating layer running alongside you.
3
Automate only the ones you run 3x per week or more
Inbox triage (WF01) and meeting notes (WF02) are the first candidates. Once you have run them manually 10+ times, connect them to n8n or Make. The prompt stays the same. The trigger becomes automatic. The rest do not need automation until they become repetitive enough to earn it.
